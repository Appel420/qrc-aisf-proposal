# Contribution Guide

The QRC-AISF project is open to all under the MIT License to advance ethical AI standards. See LICENSE.md.

## Getting Started

1. Fork the repository.
2. Clone your fork locally.
3. Create a feature branch (`git checkout -b feature/your-feature`).
4. Commit and push. Pull requests merge if tests pass.

## Submission Requirements

- Run `./validator.py` to confirm HIPAA, FISMA, and linting compliance.
- No breaking changes to encryption protocols.
- No exposed tokens or sensitive data.
- Limit network calls to documented endpoints.

## Support

Open an issue for bugs, features, or use cases (e.g., medical or government applications). The maintainer will respond.
